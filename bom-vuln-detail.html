CVE-2024-22243
Spring Framework URL 解析漏洞 — 开放重定向与服务端请求伪造 (SSRF)
HIGH
漏洞基础信息
漏洞名称Spring Framework URL Parsing Vulnerability (Open Redirect & SSRF)
查看链接https://nvd.nist.gov/vuln/detail/CVE-2024-22243
CVSS 评分8.1HIGH
CVSS 向量CVSS:3.1/AV:N/AC:L/PR:N/UI:R/S:U/C:H/I:H/A:N
CWE 类型CWE-601URL Redirection to Untrusted Site ('Open Redirect')
影响范围Spring Framework 6.1.0 - 6.1.36.0.0 - 6.0.165.3.0 - 5.3.31
解决方案https://spring.io/security/cve-2024-22243 升级至 Spring Framework 6.1.4+ / 6.0.17+ / 5.3.32+
漏洞详情Spring Framework 中 UriComponentsBuilder 在处理包含用户信息 (userinfo) 的 URL 时存在解析缺陷。当应用程序使用 UriComponentsBuilder 解析外部提供的 URL 并对解析后的主机名 (host) 组件进行安全校验时，攻击者可以通过构造包含特殊字符的 userinfo 部分，导致解析结果与实际浏览器行为不一致。此差异可被利用，绕过主机名校验，实现开放重定向 (Open Redirect) 或服务端请求伪造 (SSRF) 攻击。
